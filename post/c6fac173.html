开发一个SDK（starter）

理想情况：开发者只需要关心调用哪些接口、传递哪些参数。就跟调用自己写的代码一样简单。

开发starter的好处：开发者引入之后，可以直接在application.yml中写配置，自动创建客户端

1、新建项目

创建一个liapi-client-sdk的springboot项目 勾选lombok、Spring Configuration Processor（作用：自动生成配置的代码提示）

然后处理pom.xml 这个一定需要删除因为这个是maven的构建项目成可运行jar包。现在是制作starter依赖包

2、编写配置类

不需要spring boot启动类，将其删去。

3、指定配置类

新建resources/META-INF/spring.factories并指定

4、发布starter

双击Maven lifecycle下的install或者命令行mvn install

5、测试

引入依赖

回到liapi-Interface项目，把之前的client、util、model全部删掉。

然后在pom中引入刚制作好的starter

注意：这里能直接引入，是因为刚install的stater在我们的本地，也可以发布到Maven仓库或者提供jar包供大家使用。
